# Audio Doctor 论文白色主题瀑布图配色需求

写给 GOODMETER 施工队。本文只描述论文图例侧的视觉需求，不限定具体实现方式。

## 目标

Audio Doctor 的白色主题瀑布图需要从“软件界面里好看”调整为“放进本科论文里也稳重、清楚、可信”。目标不是做得灰、旧、脏，也不是追求炫技艺术感，而是做成接近学术图表的清爽视觉：白底、结构清楚、频段可读、能量层级明确。

一句话标准：严肃，但不能泔水；好看，但不能像海报。

## 一、适用范围

本需求优先作用于论文导出主题，例如：

- `thesis_light`
- `paper_light`
- `export_light`

不要求替换软件内所有白色主题。软件界面可以保留更艺术的配色，但论文导出需要单独稳定版本。

优先影响这些图：

- Spectrogram A/B/C
- Tail Spectrogram
- Reverb / Space 视图中的频谱残留区
- 其他类似瀑布图、时频图、能量热力图

## 二、整体视觉方向

推荐方向：

- 背景接近纯白或轻微暖白；
- 坐标轴和网格使用低饱和冷灰；
- 低能量区域保持干净浅色，不要糊成灰绿色；
- 中能量区域使用蓝、青、蓝紫一类冷色；
- 高能量区域可以进入琥珀、橙红，但只用于真正高能量；
- 文字保持深灰或近黑，保证缩小进 Word 后仍可读。

不推荐方向：

- 大面积黄绿、土黄、灰褐混合；
- 低亮度、低对比的脏灰色底；
- 过度彩虹色，像频谱插件炫技界面；
- 过饱和霓虹色；
- 红绿对比作为主要编码；
- 高能量区域一大片红，导致信息失真；
- 看起来像医学热图、天气雷达、污染图或发霉截图。

## 三、推荐色带

建议做一个专门的论文色带，不直接复用软件艺术色带。

### 推荐方案 A：Blue to Amber

适合论文主力使用，稳重且容易读。

- 低能量：`#F7FAFC`
- 弱能量：`#DCEBFA`
- 中低能量：`#8EC5E8`
- 中能量：`#2F80B7`
- 中高能量：`#4E63AC`
- 高能量：`#F2B84B`
- 极高能量：`#D95F3D`

特点：

- 蓝色负责“可读、理性、干净”；
- 琥珀/橙红只负责强调峰值；
- 不容易脏；
- 放进白底论文比较稳。

### 推荐方案 B：Viridis-like Academic

适合需要更严肃、更像科研图的场合，但要避免变成黄绿泔水。

- 低能量：`#F8FAFC`
- 弱能量：`#DDE7F0`
- 中低能量：`#7DB6B8`
- 中能量：`#2F7F95`
- 中高能量：`#355C9A`
- 高能量：`#E2B94B`
- 极高能量：`#CC6B3C`

注意：这个方案如果中段绿色过多，很容易脏。绿色只能作为过渡，不要成为主色。

### 暂不推荐

- Jet/rainbow 全彩虹；
- 黄绿棕连续渐变；
- 紫粉蓝艺术渐变；
- 深灰底加彩色热图直接反白。

## 四、背景、网格与文字

背景：

- 图区背景建议 `#FFFFFF` 或 `#FBFCFD`；
- 不要用明显米黄、灰黄、灰绿；
- 不要添加渐变背景、纹理、发光、阴影。

网格：

- 主网格：`#D6DCE3`，线宽 1px，透明度 60%-80%；
- 次网格：`#E9EDF2`，线宽 1px，透明度 40%-60%；
- 网格要帮助读数，不要抢过热力图主体。

坐标轴：

- 轴线：`#7A8490`；
- 刻度文字：`#3B424A`；
- 单位文字：`#2F363D`；
- 字号需要考虑 Word 中缩小到 12-14cm 宽后仍可读。

标签：

- 轨道标签如 `DRY A / WET A` 使用深灰或对应曲线主色；
- 不要把标签压在高能量区域上；
- 必要时给标签加白色半透明底，但不要做成大卡片。

## 五、能量映射要求

瀑布图的颜色必须服务论文分析，不能只是好看。

建议：

- 低能量区域保持接近白底，但不能完全消失；
- 中能量区域要能看出形状、走势和残留；
- 高能量区域只在峰值、瞬态、主频带附近出现；
- 相同 job 内的 A/B/C 轨使用同一色带和同一 dB 映射；
- manifest 记录色带名称、dB min/max、是否采用 percentile clipping。

建议默认映射：

- `floorDb`: `-80 dB`
- `ceilingDb`: `0 dB`
- 或使用 2%-98% percentile clipping，但 manifest 必须写清楚。

论文图不建议每张图自动单独拉伸到完全饱满，否则 A/B/C 对比会失真。

## 六、论文导出预设

建议新增一个明确的导出参数：

```json
{
  "theme": "thesis_light",
  "colormap": "blue_amber_academic",
  "background": "paper",
  "dpiIntent": "word_pdf",
  "width": 1800,
  "height": 900,
  "fixedDbScale": true,
  "floorDb": -80,
  "ceilingDb": 0
}
```

如果代码里暂时不想做这么多字段，也至少支持：

- `theme: "thesis_light"`
- `colormap: "blue_amber_academic"`
- 固定图像尺寸；
- 固定 dB 范围；
- manifest 写出本次导出主题与色带。

## 七、验收标准

施工队改完后，请至少导出三组图给论文侧验收：

1. 单个素材的 Spectrogram；
2. DRY A/B/C 三轨 Spectrogram；
3. Reverb / Space 或 Tail Spectrogram。

每张图需要同时满足：

- 放大看有细节；
- 缩到 Word 正文宽度后仍能读；
- 白底上不刺眼；
- 打印成灰度后仍能大致区分强弱；
- 低能量区域不脏；
- 高能量区域不糊成一片红；
- A/B/C 的强弱差异不是由自动拉伸造成的假对比；
- 论文里看起来像分析图，不像软件宣传图。

## 八、当前优先级

最高优先级：

- 增加 `thesis_light` 论文导出主题；
- 增加一套 `blue_amber_academic` 色带；
- 固定 A/B/C 同图的 dB 映射；
- manifest 记录色带和动态范围参数。

次优先级：

- 标签避让；
- 灰度可读性测试；
- 多尺寸导出；
- Word 插图安全边距。

暂时不急：

- 复杂 3D 瀑布；
- 可交互色带编辑器；
- 大量艺术主题；
- 完整出版级排版系统。

## 九、给论文侧的判断话术

这套白色主题不是为了“更炫”，而是为了让图例在论文中承担解释功能：读者能够直接看到能量在时间和频率上的分布、残留与变化，而不会被配色本身带偏。

最终效果应当接近“清楚的声学分析图”，而不是“漂亮的软件截图”。
